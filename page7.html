Arrow Functions
2-minute Recap (from Day 1)
Yesterday, we covered function declarations: `function name() { ... }`. They help reuse code.

New Core Concept: Arrow Functions
Arrow functions are a shorter way to write functions. Think of them like shortcuts for simple functions.

- Syntax: `const add = (a, b) => a + b;`
- Real-world analogy: A text shortcut (like "brb" for "be right back") saves time typing.

Example: const greet = () => console.log("Hi!"); greet(); // outputs "Hi!"

Function Scope and Closures
2-minute Recap (from Day 2) Yesterday, we covered arrow functions: shorter syntax for writing functions.
New Core Concept: Function Scope
Scope determines where variables are accessible. Think of it like compartments in a building - variables in one compartment (function) aren't automatically in another.
- Variables declared inside a function are local to that function.
- Real-world analogy: A secret in a room is only known to people in that room.

Example: function outer() {
let x = 10;
function inner() {
console.log(x); // accesses x from outer
}
inner();
}
outer(); // outputs 10

Closures:
A closure is when a function "remembers" its outer scope's variables.

- Example: function createGreeting(name) { return () => console.log(`Hello, ${name}!`); } const greetJohn = createGreeting("John"); greetJohn(); // outputs "Hello, John!" *Test Understanding:* 1.
